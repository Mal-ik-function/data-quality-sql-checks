# Data Quality | SQL Checks

This repository contains structured SQL validation checks designed to detect data quality issues in operational systems.

The purpose is to demonstrate how data governance and control requirements can be translated into executable queries.

---

## Areas covered

- Missing critical fields
- Duplicate records
- Referential integrity gaps
- Invalid data ranges

---

## Why this matters

Poor data quality creates:
- Reporting inaccuracies
- Regulatory risk
- Operational inefficiencies
- Customer impact

Structured validation checks allow early detection and remediation.
